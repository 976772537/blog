Spring常用注解

1.声明bean注解

@Component 通用注解

@Service 业务层

@Repository 数据访问层

@Controller 控制层

2.注入bean注解

@Autowired 根据属性注入 与 @Qualifier 一起使用

@Resource 根据名字注入

3.java配置相关

@Configuration 声明当前类为配置类，内部包含了@Component

@Bean 声明当前方法的返回值为一个bean

@ComponentScan 用于对Component进行扫描

@WishlyConfiguration 为@Configuration 与 @Component
的组合注解

4.切面（aop）相关

@Aspect 申明该类是一个切面，通过@After、@Before、@Around定义建言（advice），可直接将拦截规则（切点 pointcut） 作为参数

@After 在方法执行之后执行

@Before 在方法执行之前执行

@Around 在方法执行之后与之前执行

@PointCut 声明切点，在java的配置类中使用@EnableAspectJAutoProxy注解开启Spring对AspectJ代理的支持（类上）

Spring AOP切点表达式用法总结

5.@Bean的属性支持

@Scope 设置Spring容器如何新建Bean实例（方法上需要有@Bean），其设置类型主要包括：

  • Singleton 单例，默认模式
  • Protetype 每次调用新建一个bean
  • Request web项目中,给每个http request新建一个bean
  • Session web项目中,给每个http session新建一个bean
  • GlobalSession 给每一个 global http session新建一个Bean实例

全局Session-GlobalSession

6.@Value注解

@Value 为属性注入值（属性上）支持如下注入：

  • 普通字符注入

@Value(“aaaa”)String name;

  • 操作系统属性注入

@Value(“#{systemProperties[‘os.name’]}”)String osName;

  • 表达式结果注入

@Value(“#{ T(java.lang.Math).random() * 100 }”) String randomNumber;

  • bean属性注入

@Value(“#{domeClass.name}”)String name;

  • 文件资源注入

@Value(“classpath:com/hgs/hello/test.txt”)String Resource file;

  • 网站资源注入

@Value(“http://www.cznovel.com")Resource url;

  • 配置文件注入

@Value(“${book.name}”)String bookName;

@Value详解

@EnableAsync 配置类中，通过次注解开启对异步任务的支持，叙事性AsyncConfigurer接口（类上）

@Async 在实际执行的bean方法使用该注解来声明其是一个异步任务（需要@EnableAsync注解)

@EnableAsync 异步调用

@RunWith
用于对Junit的支持@RunWith(SpringJUnit4ClassRunner.class)

ContextConfiguration 用来加载配置ApplicationContext，其中classes属性用来加载配置类

@ContextConfiguration(classes={TestConfig.class})

@ContextConfiguration的意思

SpringMVC

@EnableWebMvc 在配置类中开启Web MVC的配置支持

@EnableWebMvc使用坑点解析

@Controller 声明该类为SpringMVC中的Controller

@RequestMapping 用于映射Web请求，包括访问路径和参数

@ResponseBody 支持将返回值放在response内，而不是一个页面，通常用户返回json数据

@RequestBody 支持将参数放入request体中，而不是直接连接在地址后面。

@PathVariable 用于接收路径参数，比如：@RequestMapping(“/hello/{name}”)声明的路径，将注解放在参数前，即可获得该值，通常作为Restful的接口实现方式。

@RestController 该注解为一个组合注解，相当于@Controller和@ResponseBody的组合，相当于该@Controller的所有方法都默认加上了@ResponseBody。

@ControllerAdvice 通过该注解，我们可以将对于控制器的全局配置放置在同一个位置，注解了@Controller的类的方法可使用@ExceptionHandler、@InitBinder、@ModelAttribute注解到方法上， 这对所有注解了 @RequestMapping的控制器内的方法有效。

@ExceptionHandler 用于全局处理控制器里的异常

@InitBinder 用来设置WebDataBinder，WebDataBinder用来自动绑定前台请求参数到Model中。

@ModelAttribute 本来的作用是绑定键值对到Model里，在@ControllerAdvice中是让全局的@RequestMapping都能获得在此处设置的键值对。
